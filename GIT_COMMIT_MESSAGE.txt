## Unified Credentials System

Both versions now use **rh00d.sct** (JSON format):
```json
{
  "api_key": "rh.your_key",
  "private_key": "base64_encoded_seed"
}
```
- Single file consolidates r_key.txt + r_secret.txt
- Automatic permission enforcement (0600 owner-only)
- Python GUI wizard generates securely
- C version parses with safe JSON extraction

## Security Hardening

✅ Permission enforcement: Fails fast if exposed (0600 required)
✅ Atomic file writes: Temp file + rename + chmod pattern
✅ Buffer safety: All strings bounds-checked (snprintf, strncpy)
✅ No shell injection: Direct POSIX APIs (mkdir, chmod, rename)
✅ No hardcoded secrets: All read from external files
✅ Error handling: Clear messages guide user on issues

Security Rating: A (suitable for production testing)
